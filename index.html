A Kubernetes-native infrastructure management platform for bare metal servers

Welcome to the Metal3

About Metal³


Metal³ is a Free and Open Source Software (FOSS) community that aims to simplify the deployment and management of bare metal infrastructure in a Kubernetes environment. It provides a declarative way to manage bare metal servers using the Kubernetes API, allowing users to provision and manage their bare metal infrastructure in a scalable, reliable, and automated way.


Project


The Metal³ project is part of the Kubernetes project and is a subproject of the Cluster API project. The project's mission is to provide a declarative API-driven approach to bare metal infrastructure management that is consistent with the Kubernetes API and ecosystem. Metal³ is backed by Ironic, a component from the OpenStack ecosystem, which is responsible for booting and installing machines. Metal³ handles the installation of Ironic as a standalone component and can provision machines regardless of the brand of hardware. Ironic is supported by a mature community of hardware vendors and supports a wide range of bare metal management protocols.


Goal


The goal of Metal³ is to provide a Kubernetes-native infrastructure management platform for bare metal servers. This means that Metal³ aims to make it easy to manage and deploy Kubernetes clusters on bare metal servers by providing tools and components that integrate with Kubernetes resources and APIs. By doing so, Metal³ eliminates the need for separate management tools for bare metal infrastructure, making it easier to manage and maintain Kubernetes clusters on bare metal. The project also aims to be open-source and community-driven, encouraging contributions and collaboration from a wide range of developers and users.


Mission


The mission of the Metal³ project is to simplify the management and deployment of Kubernetes clusters on bare metal servers. The project aims to achieve this mission by providing a Kubernetes-native infrastructure management platform that can enroll bare metal machines, provision operating system images, and deploy Kubernetes clusters to them. Metal³ also supports upgrading Kubernetes or the operating system in the clusters with a non-disruptive rolling strategy and automatically remediating failed nodes. Metal³ is an open-source project that welcomes community contributions and strives to be inclusive and diverse. Ultimately, the mission of Metal³ is to make it easier for developers and operators to manage and maintain their bare metal infrastructure, and to empower organizations to run Kubernetes on their own hardware.


Components


This project mainly consists of several components. The main components are:

  • Bare Metal Operator: The Bare Metal Operator is a Kubernetes operator that manages the lifecycle of bare metal hosts. It can discover hardware inventory, configure BIOS and RAID settings, install and boot an operating system image of choice, and deploy Kubernetes.
  • Cluster API: The Cluster API Provider Metal³ is a Kubernetes provider that integrates the Cluster API sub-project with Metal³ components. It enables the creation and management of Kubernetes clusters on bare metal servers using the Cluster API.
  • Ironic: Ironic is a standalone component from the OpenStack ecosystem that Metal³ uses for booting and installing machines. Metal³ handles the installation of Ironic as a standalone component, making it easy to use without having to bring along the rest of OpenStack.
  • Bare Metal-operator: The bare metal-operator is a Kubernetes controller that automates the process of discovering and managing bare metal infrastructure. It can manage hardware inventory, perform RAID configuration, and provision and install operating systems.
  • Tempest-plugin: The Metal³ Tempest plugin is used.
